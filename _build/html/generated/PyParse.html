Functions

buildHTML(save_dir, compoundDF, all_compounds)

Build a HTML output file line by line with links to all plots generated, and output key columns of the compoundDF.

convertRPT2Dict(filename)

Converts the input .rpt file into a dict where each dict index corresponds to a well.

findHits(compound, dataTable)

For a given compound, look at each peak in each well to find a suitable match based on m/z data.

findOverlap(dataTable, well, time)

Provided with the dataTable and a retention time+well of interest, this function will look for any overlapping peaks, and return the retention times of those peaks to the user.

findPotentialConflicts(compoundDF)

Find any products which are in danger of overlapping with other compounds that are expected in the same well.

generateMol(smiles, name, save_dir)

Generates a 2D rendering of the given structure and saves it as a .png

generateOutputTable(compoundDF, internalSTD, ...)

Reformats the validated hits into a pandas table ready for visualisation and export.

getChromatogram(spectrum)

Takes the region of text in rpt file corresponding to the chromatogram, and extracts the data into two lists, corresponding to x-values and y-values.

getMSData(spectrum)

Takes the specific region of the rpt file pertaining to m/z data for a specific peak in a specific well, and returns a 2-D list containing all m/z peaks and their normalised intensity.

getUVData(spectrum)

Takes the specific region of the rpt file pertaining to UV absorbance spectrum data for a specific peak in a specific well, and returns a list containing all the maxima of that spectrum.

getUserReadableWell(wellno)

Converts the well as a number into a user-friendly string, e.g.

importStructures(filename)

From a given CSV, with defined header names, and one row per well, deconvolute the information to give a dataframe of one compound per row, indexed by canonical SMILES.

main()

Provide an .rpt file and a csv containing the compounds in the plate to analyse the wells and return an output containing plots of compounds, key wells, and multiple output tables.

plotChroma(cpname, wellno, trace, pStart, ...)

Plots the LCMS trace with labels for compounds found for a specific well, and highlights a particular peak of interest, providing m/z data for that peak.

plotDonut(dataframe, save_dir)

Takes the outputTable dataframe and generates a donut chart for product %area to be saved in the output folder.

plotHeatmap(zvalue, outputTable, save_dir)

Plots and saves a heatmap given the full dataset and the parameter to plot by, e.g.

plotHistogram(dataframe, save_dir)

Takes the outputTable dataframe and generates a histogram for product %area to be saved in the output folder.

plotHitValidationGraph(cpname, ...)

Plots all the hit peaks in a scatter graph of peaktime vs well, colour coded by whether the hit was included or discarded from the final output.

plotPieCharts(zvalue, outputTable, save_dir, ...)

Plots a set of pie charts for the full plate using the full dataset.

refineClusterByMassConf(cluster, comments)

Takes in input cluster of all the hit peaks, and refines them by ensuring all peaks have a similar mass confidence to the cluster's mean.

refineClusterByTime(cluster, comments, ...)

Takes in input cluster of all the hit peaks, and refines them by finding a mid-value for the retention time based on which hit has the greatest number of nearest neighbours.

refineClusterByUV(cluster, UVdatafound, comments)

Takes in input cluster of all the hit peaks, and refines them by ensuring all peaks have a "sensible" set of UV maxima.

removeDupAssigns(compoundDF, internalSTD, ...)

Checks each compound to ensure that no peak has been assigned to two different compounds.

selectClusterByMassConf(clusters)

If more than one cluster was found for the compound, this function is called to try to select a single cluster based on which cluster has the highest mean massConf.

selectClusterBySize(clusters)

If more than one cluster was found for the compound, this function is called to try to select a single cluster based on which cluster is the largest.

validateHits(cpname, peakList, expected_rt)

Looks at all hits for a given compound, and refines the list based on retention time, massConf and UV data.
